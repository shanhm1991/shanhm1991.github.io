windows安装mysql

words: 371    views:    time: 1min
日常笔记

环境：win7_x64
版本：mysql-5.7.18

1、ERROR 1045 (28000): Access denied for user ‘root’@’localhost’

解决：跳过密码登录mysql，修改root密码后重新登录

2、sql执行错误#3167

解决：set global show_compatibility_56=on;

场景：官网下载mysql-5.7.18-winx64包，解压，设置好环境变量，执行install等命令安装完毕，都正常未发现什么问题。但是使用客户端连接数据库时报错：ERROR 1045 (28000): Access denied for user ‘root’@’localhost’ …

1、在mysql的home目录下找my.ini文件，如果没有可以自己新建，基本配置差不多，主要在最下面添加一行：skip-grant-tables

2、重启mysql服务，然后重新登录

3、这时可以执行sql了，于是直接更新user表，注意字段名不叫password，而是authentication_string，可以select确认下

于是：

4、修改后quit，然后用密码重新登录（到这里已经解决了root的密码问题），接着新建库

5、新建用户时，注意要去掉my.ini中的skip-grant-tables，然后重启服务，不然会报错，建完之后设置密码并给用户赋权，最后执行：flush privileges（将修改刷到内存中，实时生效）

6、最后用客户端连接，发现报错：sql执行错误#3167

网上查下错误3167，大概意思是mysql5.7.6开始information_schema.global_status已经被舍弃，为了兼容性，此时需要打开show_compatibility_56，即：set global show_compatibility_56=on; 修改之后就可以正常连接使用了。
